Protocols

HTTP methods

GET - the method is used to retrieve a representation of a resource from the server.

POST - the method is used to submit data to the server to create a new resource.

PUT - the method is used to update or replace an existing resource on the server.

PATCH - the method is used to partially update an existing resource on the server.

DELETE - the method is used to remove a resource from the server.

HTTP status codes

  • 1xx: Informational responses
  • 2xx: Successful responses
  • 3xx: Redirection messages
  • 4xx: Client error responses
  • 5xx: Server error responses

Examples:

  • 100 - Continue
  • 201 - Created
  • 310 - Moved permamentaly
  • 404 - Not found
  • 500 - Internal server error

DNS list of types

  • A (Address) Record: This record maps a domain name to an IPv4 address. It is used to translate human-readable domain names into machine-readable IP addresses.
  • AAAA (IPv6 Address) Record: Similar to the A record, the AAAA record maps a domain name to an IPv6 address. It is used for translating domain names to IPv6 addresses.
  • CNAME (Canonical Name) Record: The CNAME record creates an alias for a domain name and points it to another domain name. It is commonly used to map subdomains or alternative domain names to the primary domain.
  • MX (Mail Exchanger) Record: The MX record specifies the mail server responsible for handling incoming email for a domain. It defines the domain name of the mail server and its priority for handling mail delivery.
  • TXT (Text) Record: The TXT record is used to store arbitrary text data associated with a domain. It is commonly used for various purposes such as domain verification, SPF (Sender Policy Framework) records for email authentication, and providing additional information about a domain.
  • NS (Name Server) Record: The NS record specifies the authoritative name servers for a domain. It identifies the DNS servers that are responsible for serving DNS records for a particular domain.
  • PTR (Pointer) Record: The PTR record is used in reverse DNS lookups. It maps an IP address to a domain name, allowing you to find the domain associated with a given IP address.
  • SRV (Service) Record: The SRV record is used to specify the location of a specific service within a domain. It provides information about the hostname, port, priority, weight, and other details of a particular service.
  • SOA (Start of Authority) Record: The SOA record provides authoritative information about a DNS zone. It includes details such as the primary name server for the domain, the contact email address of the domain administrator, zone serial number, and other configuration parameters.
  • SPF (Sender Policy Framework) Record: The SPF record is used to specify the authorized sending mail servers for a domain. It helps prevent email spoofing and verifies the authenticity of the email sender.

Identify Google's IP

To identify google.com.ua IP, follow the next steps:

  1. Open Terminal
  2. Run the command nslookup google.com.ua

As a result, you will receive "name" and "address":
Name: google.com.ua
Address: 216.58.215.99
